## 制作意図

「ゲームに興味がない人にも情報が自然と目に入る」設計を意識しました。

クリックという能動的な操作を前提にせず、マウスオーバーするだけで右パネルの情報が切り替わります。  
PCとスマホでは操作の特性が異なるため、インタラクションを分けて設計しています。

---

## 機能一覧

### PC（1024px以上）
- マウスオーバーで右パネルに詳細情報を表示
- カードクリックでパネルを固定（同じカードを再クリックで解除、Escでも解除）
- 右パネルの背景画像をクロスフェードでスライドショー切り替え

### スマホ（1023px以下）
- カードタップで全画面モーダルを表示
- 下スワイプでモーダルを閉じる（iOS風）

### 共通
- タイトル名で絞り込める検索バー（debounce処理つき）
- プラットフォームフィルタータブ（全て / PS / Switch / app）
- スクロールで検索バーとタブが画面上部に固定
- アクセシビリティ対応（スキップリンク・ARIA属性・フォーカストラップ・キーボード操作）

## ファイル構成

```
index.html   ページ構造
style.css    スタイル
script.js    インタラクション全般
games.js     ゲームデータ（タイトルを追加・編集する場合はこのファイルのみ変更）
img/         画像ファイル
```

ゲームデータは `games.js` に分離しています。タイトルの追加・変更は `GAMES` 配列を編集するだけで、HTMLやJSを触る必要はありません。

## 工夫した点

**設計面**
- 非ゲーマーをターゲットにした「パッシブ露出」の設計（マウスオーバーで情報が目に入る）
- PCとスマホで操作方法を分けた設計（PC：マウスオーバー＋固定、スマホ：タップ＋モーダル）

**実装面**
- 検索のdebounce処理（入力のたびに処理が走らないよう150ms待機）
- スライドショーの2枚構成によるクロスフェード
- スワイプ中の閾値判定とバイブレーションフィードバック
- `matchMedia` によるPC・スマホ判定のキャッシュ（パフォーマンス最適化）

**アクセシビリティ**
- キーボードのみで全操作が可能
- モーダル開閉時のフォーカストラップとフォーカス復元
- スクリーンリーダー向けARIA属性
